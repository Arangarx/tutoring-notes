Mynk — round 5: the full matrix
4 palettes × 4 typography combos · 16 cells · 2026-05-19

Every palette × every typography combination, in one grid.

Round 4 showed F1–F4 with one shared typography choice, and the typography matrix in isolation against F1 only. You wanted to see all the cross-cuts. So: 4 rows (the four palette softness variants) × 4 columns (V3w+V2h, V3w+V3h, V4w+V2h, V4w+V3h) = 16 specimens.

Each cell is intentionally compact — wordmark, heading, sub, body, CTA, AI tag, recording dot. Enough to read the typography and palette together without the visual noise of a full mocked dashboard. Toggle light/dark for the dark-mode pass.

How to use this: First scan vertically — which palette feels right across all four typography variants? Then scan horizontally — which typography combination holds up across all four palettes? The winner is the cell whose row and column both feel like the right answer. If you can't find one, the constraint is in either the palette set or the typography set, not in the matrix.

How to use the matrix to land somewhere

The matrix isn't trying to pick a winner — it's giving you 16 anchored data points so each family member can vote on the one cell that holds together best. Suggested workflow:

  • Step 1 — each person picks a row. Just stare at each row left-to-right. Which palette do you find your eye returning to across all four typography variants? That row is your palette pick. If everyone picks a different row, you've found the real disagreement and it's not about typography.
  • Step 2 — each person picks a column. Same idea vertically. Which typography combo holds up across all four palettes? That's the typography pick. If two columns tie, switch back to row mode and see which combo plays best with the row you already picked.
  • Step 3 — confirm the intersection. Your row × column = your cell. Look at just that cell and ask: "is this Mynk?" If yes, ship it. If no, the constraint is in one of your other choices — try a different row or column and see if the intersection feels more right.

My priors after building this: F2 × V4w+V2h and F1 × V4w+V2h are the two strongest cells. The V4 wordmark + V2 heading column generally holds up best across palettes (soft mark, crisp message — clean hierarchy). F1 is the most distinctive palette; F2 is the calmer palette daughter will likely prefer. F3 sometimes reads as "F1 with a sleepy CTA." F4 sometimes reads as "generic calm SaaS." But trust the family vote over my priors — the matrix is the actual deliverable here.
