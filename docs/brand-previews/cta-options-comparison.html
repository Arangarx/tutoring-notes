Cream #FCFBF4 on coral #E27D60 = 2.76:1 — FAILS WCAG SC 1.4.3

The current light-mode CTA combination is unreadable by WCAG's threshold (need 4.5:1 for normal text). This affects every .btn-primary in the app and any future button using the coral fill. Each option below renders three real surfaces — marketing hero, dashboard with the up-next card, and pre-session preview — using that option's tokens. Coral persists for live-dot, AI summary tints, eyebrows, and mic-ring in every option; the only thing that changes is the CTA fill and/or text. Toggle dark mode in the header to verify dark-mode parity.

Where coral lives in each option.

Skim the four blocks below side-by-side. Same content in each. Pay attention to: (1) how the eye moves to the primary CTA, (2) whether the live-dot + AI-summary panel still feel like the brand's "spark," (3) whether the mood goes too austere (lose energy) or too soft (lose hierarchy).

Option A — dark text on coral. Token-cheapest fix. Coral keeps the CTA role; the text on top just goes dark. 5.64 AA
Option B — burnt-coral CTA fill (--accent-text becomes the fill). Cream text stays. Cleanest token-wise; mood goes from bright-coral to deep-burnt-coral. 7.35 AAA

How to use this page

Toggle dark mode in the header and re-scan all four options. Dark mode passes WCAG for every option (the audit's blocker is light-mode-only) — the dark renderings are here to confirm none of the options regress dark-mode parity.

What to compare:

  • The marketing hero CTAs ("Try it for one session") side-by-side — which feels most clickable without going austere?
  • The dashboard up-next card — does the CTA still pop against the brand-blue card? Option D specifically: the brand-blue CTA on a brand-blue card needs a Phase 1 design fix (invert the card, or use a stronger affordance).
  • The pre-session "Start session" — does it still feel like the obvious next thing?
  • The live-dot, eyebrow text, AI summary panel tint — all four options preserve coral here. Verify the brand still has its "spark."

What this is not: the final design. This is just the four palette options applied to existing layouts. The actual Phase 1 design specs come after this decision; they may further evolve the layouts (e.g., the up-next card under Option D).
